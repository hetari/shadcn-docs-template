This guide walks you through every touchpoint you need to personalize this template — from navigation links to the command menu search palette.

## Site Configuration

The central place for branding and links is `app/lib/siteConfig.ts`. Update these values **first**:

```ts
// app/lib/siteConfig.ts
export const siteConfig: SiteConfig = {
  name: "My Project", // shown in header & OG tags
  url: "https://my-project.dev",
  ogImage: "https://my-project.dev/og.jpg", // social sharing image
  description: "A short description of your project.",
  links: {
    github: "https://github.com/you/my-project",
    githubAPI: "https://api.github.com/repos/you/my-project",
    ungh: "https://ungh.cc/repos/you/my-project",
  },
};
```

## Navigation Configuration

Navigation in this template is controlled through constants in `app/lib/navigation.ts`. Understanding these constants will help you customize both desktop and mobile navigation.

### Navigation Constants Overview

All navigation configuration is centralized in `app/lib/navigation.ts`. Here's what each constant controls:

```ts
// app/lib/navigation.ts
export type NavItem = {
  name: string;
  href: string;
};

/** Main navigation links shown in desktop header and mobile menu */
export const MAIN_NAVIGATION: NavItem[] = [
  { href: "/docs/getting-started/introduction", name: "Docs" },
  { href: "/docs/prose-demo", name: "Demo" },
];

/** Section pills shown in sidebar and mobile menu */
export const NAV_SECTIONS: NavItem[] = [
  { name: "Get Started", href: "/docs/getting-started/introduction" },
];

/** Sidebar group titles (lowercased) that should never be rendered */
export const SIDEBAR_EXCLUDED_SECTIONS: string[] = [];

/** Individual page paths that should be hidden from the sidebar */
export const SIDEBAR_EXCLUDED_PAGES: string[] = [
  "/docs/introduction",
];
```

| Constant                    | Controls                                                      | Used By                                 |
| --------------------------- | ------------------------------------------------------------- | --------------------------------------- |
| `MAIN_NAVIGATION`           | Main navigation links in header                               | `MainNav.vue`, `MobileNav.vue`          |
| `NAV_SECTIONS`              | Section pills in sidebar and mobile menu                      | `DocsSidebar.vue`, `MobileNav.vue`      |
| `SIDEBAR_EXCLUDED_SECTIONS` | Hides entire sidebar groups by title (case-insensitive)       | `DocsSidebar.vue`                       |
| `SIDEBAR_EXCLUDED_PAGES`    | Hides specific pages from sidebar page list                   | `DocsSidebar.vue`                       |

## Desktop Navigation

**File:** `app/components/MainNav.vue`

The desktop navigation displays horizontal navigation buttons in the header. It receives navigation items via props and does **not** need to be edited directly.

---

## Mobile Navigation

**File:** `app/components/MobileNav.vue`

The mobile navigation displays a hamburger menu that opens a full-screen overlay with three sections of links.

### Mobile Menu Structure

The mobile menu displays **three sections**:

#### 1. Menu Section (Main Links)

**Source:** `MAIN_NAVIGATION` constant  
**Location:** First section in mobile menu  
**Contains:**

- "Home" link (hardcoded to `/`)
- All items from `MAIN_NAVIGATION` array

#### 2. Sections Section

**Source:** `NAV_SECTIONS` constant  
**Location:** Second section in mobile menu  
**Contains:** Section pills (e.g., "Get Started", "Components")

::callout{type="warning"}
To hide MCP documentation sections, set `showMcpDocs` to `false`.
::

#### 3. Content Tree Section

**Source:** Auto-generated from `content/docs/` directory structure  
**Location:** Third section in mobile menu  
**Contains:** All documentation pages organized by folder structure

## Navigation Constants Explained

### `MAIN_NAVIGATION`

**Purpose:** Main navigation links shown in the header  
**Shows in:**

- Desktop navigation (`MainNav.vue`)
- Mobile menu "Menu" section
- Command menu navigation items

### `NAV_SECTIONS`

**Purpose:** Section pills displayed in sidebar and mobile menu  
**Shows in:**

- Desktop sidebar (top section)
- Mobile menu "Sections" section

::callout{type="note"}
Sections with `/mcp` in the href are filtered by the `showMcpDocs` flag.
::

---

### `SIDEBAR_EXCLUDED_SECTIONS`

**Purpose:** Hides entire sidebar groups by their title  
**Shows in:** Desktop sidebar only  
**Behavior:** Case-insensitive matching against group titles

**Example:**

```ts
export const SIDEBAR_EXCLUDED_SECTIONS: string[] = [
  "installation",  // Hides "Installation" group
  "dark mode",     // Hides "Dark Mode" group
]
```

**To hide a section:** Add the section title (lowercase) to this array

::callout{type="warning"}
This only hides the group title and its children from the sidebar. The pages are still accessible via direct URL and will appear in the mobile menu.
::

---

### `SIDEBAR_EXCLUDED_PAGES`

**Purpose:** Hides specific pages from the sidebar page list  
**Shows in:** Desktop sidebar only  
**Behavior:** Exact path matching


**To hide a page:** Add the exact page path to this array

::callout{type="note"}
This is useful for hiding pages that are still relevant but might clutter the sidebar
::

::callout{type="warning"}
Use this to hide pages that are still relevant but might clutter the sidebar (e.g., an introduction page that's already linked in the header).
::
